Accepted File Formats

    Grid:
  • Tab separated with extensions .txt, .dat, .tsv, or .tab
    Image:
  • A JPEG image with extension .jpg or .jpeg
  • A PNG image with extension .png
  • A TIFF image with extension .tif or .tiff

Data Structure

There are two acceptable formats for the grid data, long-format or wide-format:
Long-Format
Grid input with strings in the first row are assumed to be in long-format. The data must contain 3 columns; X, Y, and Value. The X and Y columns correspond to the X and Y coordinates respectively. Each combination of X and Y coordinates must be in the data set. The Value column provides the value for the set of coordinates. The first row must contain the names for each column; "x", "y", and "value".
Wide-Format
Grid input with numbers in the first row are assumed to be in wide-format. Values are expressed as a matrix instead of explicitly stating each x and y coordinate combination. The data must contain at least 2 rows and 2 columns. All data corresponds to implied grid coordinates, with (1,1) as the bottom left corner of the matrix.

Range

X: positive numbers ranging from 1 to the number of grid columns
Y: positive numbers ranging from 1 to the number of grid rows
Value: any positive number
